# OctoAcme Project Management Documentation

Welcome to the OctoAcme Project Management documentation! This collection of guides provides a comprehensive framework for running projects at OctoAcme, from initial concept through delivery and continuous improvement.

## About OctoAcme

OctoAcme is a fictional company used for training and documentation purposes. These project management practices are designed to help teams deliver customer value through iterative, well-coordinated project execution.

## Who Should Use This Documentation

- **New Team Members**: Get up to speed quickly on how we run projects
- **Project Managers**: Reference guides for managing delivery, risks, and communications
- **Product Managers**: Understand the project lifecycle and key touchpoints
- **Developers**: Learn about project workflows, quality standards, and team rhythms
- **Stakeholders**: Understand how projects are structured and communicated

## Key Principles

All OctoAcme projects follow these core principles:

- **Customer-first**: Prioritize customer value and usability
- **Iterative delivery**: Deliver small, testable increments
- **Clear ownership**: Each project has named PM and Product Lead
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback and learning
